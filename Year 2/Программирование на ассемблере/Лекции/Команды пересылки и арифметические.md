При описании необходимо указывать, какие операнды допустимы, какие - нет

| Местонахождение операнда   | Обозначение  | Запись а ASM       |
| -------------------------- | ------------ | ------------------ |
| В команде                  | i8, i16, i32 | Константа          |
| В регистре общ. назначения | r8, r16      | имя регистра       |
| В сегментном регистре      | sr           | CS, DS, SS, ES     |
| В памяти                   | m8, m16, m32 | адресное выражение |
|                            |              |                    |
```nasm
mov op1, op2
```
По этой команде на место `op1` пересылается значение `op2`. Флаги команда не меняет

| op1         | op2               |
| ----------- | ----------------- |
| r8          | i8, r8, m8        |
| m8          | i8, r8            |
| r16         | i16, r16, m16, sr |
| sr (non-CS) | m16, r16          |
| m16         | i16, r16, sr      | 
Запрещено пересылать из одной ячейки памяти в другую, из одного сегментного регистра в другой, записывать непосредственный операнд в сегментный регистр  
Размер пересылаемой величины определяется по типу операндов, указанных в команде. Если известны размеры обоих операндов, то эти размеры должны совпадать  
Оператор указания типа `ptr`
```nasm
BYTE PTR <address>
WORD PTR <address>
DWORD PTR <address>
```
Используется:
1. Когда типы операндов не указаны явно
2. В регистре находится адрес ячейки и его надо обнулить
3. Необходимо изменить тип указанной переменной
Перестановка двух операндов
```nasm
XCHG op1, op2
```
Математика
```nasm
ADD
SUB
INC ;++, не меняет CF
DEC ;--, не меняет CF
NEG ; рассматривает операнд как число со знаком и меняет его. Если операнд = 128, он не меняется, OF = 1
ADC; сложение с учётом переноса, +CF
SBB; вычитание с учётом переноса 
MUL; умножение без знака
IMUL; умножение со знаком
DIV
IDIV
```
Меняются флаги: CF,OF,SF,ZF
Операнд может быть либо в регистре, либо в памяти, но не может юыть непосредственным операндом.  
Местоположение другого операнда фиксированно и явно в команде не указывается. При умножении байтов первый множитель берётся из AL. Под результат отводится в два раза больше места, чем под сомножитель.  
При умножении байтов результат имеет размер слова и записывается в AX, при умножении слов результат имеет размер двойного слова и записывается в DX/AX
